        # 2018.1.11声讯日报改造梳理

        # 1.目前可见问题：

        a)分渠道订单维度按键、接通不准，因为一个订单可能使用多个渠道呼叫，会让总分母增多，导致统计值低

        b)天润三条大线路，并没有细化到线路

        c)缺少总体统计

        d)缺少业务指标，业务指标制定还停留在去年

        粒度粗，观念旧。

        # 2.优化思路：

        解决统计错误

        粒度细化到线路

        丰富指标

        增加总览

        # 3.优化方案：

        线路分模板层面监控：去除订单维度按键率、接通率统计（因为一个订单可能多个渠道呼叫），仅统计通话维度接通率、按键率、有效按键率、重听率、错误分布。

        总览日报（分模板）：电话维度：总体接通率、按键率、有效按键率、重听率、错误分布。 订单维度：接通率、按键率、净接通率、净按键率、反馈时间、通话时长、平均呼叫次数

        KPIFunctionRemark
        接通率接通电话总数/呼叫总数
        按键率有按键电话总数/接通总数
        有效按键率有效按键电话总数/接通总数
        重听率重听次数/接通总数
        错误分布失败原因占比分布
        订单维度接通率存在>=1接通电话的订单/总处理订单数
        订单维度净接通率存在>=1接通电话的订单/总处理订单数 - 其他渠道接单的订单
        订单维度按键率存在>=1按键的订单/总接通订单数
        订单维度净按键率存在>=1按键的订单/总接通订单数不包含其他渠道接单的订单
        反馈时间sum(最后一次结束时间 - 第一次外呼时间) / 总订单数反应用户侧响应时间
        通话时长sum(结束时间-接听时间) / 按键订单总数反应商家侧效率
        平均呼叫次数总呼叫次数 / 总订单数
