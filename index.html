# 李华卿 男 92年 本科 北科大信息安全专业

# 工作经历

# 2014.06~2015.03 北京中视广信科技有限公司

独立承担歌华有线智能引擎项目的海量数据采集、数据服务接口等工作，配合各业务系统联调。同时参与大数据分析数据可视化功能的设计与开发。

# 2015.04~2016.11 多洗

负责多洗微信公众号、APP、门店、工厂、BI、客服整个O2O供应链整体项目的产品设计，架构搭建，带领团队开发、迭代，同时负责团队管理、考核等。
后端使用微服务架构，dubbo做SOA，springMVC+mybatis+RESTFUL做敏捷开发，ELK做数据实时处理，tomcat做服务器中间件，nginx做负载均衡。
前端使用vue和backbone做大型SPA，自动化编译混淆发布。APP更多使用H5，通过自定义协议调用nativeAPI。
数据存储：MySQL做元数据存储，MongoDB做流水、行为日志存储，Redis做数据缓存，使用OSS或CDN做为云端存储，减少高峰时期读取图片等资源文件的带宽压力。
使用Confluence进行内部文档管理，Jira进行项目管理，gitlab进行代码管理

# 2016.12~至今 美团

独立负责餐饮预订订单商家侧触达与交易系统中台化设计与实施.
1.解耦触达策略与触达渠道,使用模板化配置的方式和任务树的方式,优雅的解决了触达策略多变的问题,实现了触达策略的自学习.
2.独立构建声讯IVR外呼平台,模板化的方式解决多业务接入,丰富的重试规则提高触达率,同时有细粒度完善的监控帮助商家自学习声讯线路,自解决线路不稳定不适应等问题.
3.对接多个触达渠道:开店宝,美味不用等,小秘书,云管家等10多种渠道,使用适配器模式来适配参数,解决参数不同的问题.使用消息实现流程解耦,以解决流程不同的问题.
4.建立ERP直连系统,与多个ERP厂商对接,提供订单查询操作及推送服务.
5.为实现预订业务快速试错,为了快速支持预订+X模式(预订+点餐 预订+打车 预订+团购等),建立预订交易中心,实现中台化.思路是梳理各个业务流程特点,交易中心维护主流程和订单数据,提供支撑域.以接口形式将X的差异环节提供给X子模块或其他业务完成(如扣减资源,下X单等).
